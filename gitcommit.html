git commit

Utilité

git commit enregistre les modifications dans le dépôt.

Commande

    
git commit [-a | --interactive | --patch] [-s] [-v] [-u(mode)] [--amend]
           [--dry-run] [(-c | -C | --fixup | --squash) (commit)]
           [-F (fichier) | -m (msg)] [--reset-author] [--allow-empty]
           [--allow-empty-message] [--no-verify] [-e] [--author=(auteur)]
           [--date=(date)] [--cleanup=(mode)] [--[no-]status]
           [-i | -o] [--pathspec-from-file=(fichier) [--pathspec-file-nul]]
           [-S[(idclé)]] [--] [(pathspec)…​]

Description

Crée un nouveau commit contenant le contenu actuel de l’index et avec le message de validation décrivant la modification. Le nouveau commit est un fils direct de HEAD, habituellement au sommet de la branche actuelle et la branche est mise à jour pour pointer dessus.

Options

  • -a
    -all

    Indique à la commande d’indexer automatiquement les fichiers qui ont été modifiés ou supprimés, mais les nouveaux fichiers dont vous n’avez pas signalé la présence à Git ne sont pas affectés.

  • -p
    --patch

    Utilise la sélection interactive de correctif pour choisir quelles modifications valider. Référez-vous à git add pour plus de détails.

  • -C (commit)
    --reuse-message=(commit)

    Prend un objet commit existant et réutilise son message de validation et son information d’auteur (y compris l’horodatage) pour la création du commit.

  • -c
    --reedit-message=(commit)

    Comme -C, mais avec -c, l’éditeur est appelé pour permettre à l’utilisateur de modifier le message de validation.

  • --branch

    Montre la branche et l’information de suivi, y compris en format court.

  • -m (message)
    --message=(message)

    Utiliser le (message) fourni comme message de validation. Si plusieurs options -m sont fournies, leurs valeurs sont concaténées comme paragraphes séparés.

    L’option -m est incompatible avec -c, -C ou -F.
