git pull

Utilité

git pull rapatrie et intégre un autre dépôt ou une branche locale.

Commande

    
git pull [(options)] [(repository) [(refspec)…​]]

Description

La commande git pull incorpore les changements d'un dépôt distant dans la branche courante. Si la branche courante est en retard par rapport à la branche distante, alors par défaut elle avancera rapidement la branche courante pour correspondre à la branche distante. Si la branche courante et le dépôt distant ont divergé, l'utilisateur doit spécifier comment réconcilier les branches divergentes avec --rebase ou --no-rebase.

git pull est l’abréviation de git fetch suivi de git merge FETCH_HEAD.

Options

  • -q
    --quiet

    Ceci est transmis à la fois au git-fetch sous-jacent pour éliminer les rapports lors du transfert et au git-merge sous-jacent pour éliminer les rapports lors de la fusion.

  • --commit
    --no-commit

    Effectue la fusion et valider le résultat. Cette option peut être utilisée pour passer outre --no-commit.

  • -e
    --edit
    --no-edit

    Avant de procéder à une fusion automatisée réussie, lance un éditeur pour modifier le message de fusion généré automatiquement, afin que l’utilisateur puisse expliquer et justifier la fusion. L’option --no-edit peut être utilisée pour accepter le message généré automatiquement.
