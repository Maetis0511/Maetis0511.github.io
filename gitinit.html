git init

Utilité

git init permet d'initialiser un dossier git ou réinitialiser un dossier git existant.

Commande

    
git init [-q | --quiet] [--bare] [--template=(template-directory)]
         [--separate-git-dir (git-dir)] [--object-format=(format)]
         [-b (branch-name) | --initial-branch=(branch-name)]
         [--shared[=(permissions)]] [(directory)]

Description

La commande git init crée un dépôt Git vide, cela créer le répertoire .git avec des sous-répertoires pour y mettre tous les fichiers nécessaires au bon fonctionnement du répertoire git. Une branche initiale sans commit est aussi créee.

Options

  • -q
    --quiet

    Affiche uniquement les messages d'erreur et d'avertissement.

  • --bare

    Crée un dépôt nu. Si l'environnement GIT_DIR n'est pas défini, il est défini comme le répertoire de travail actuel.

  • --object-format=(format)

    Spécifie le format d'objet donné (algorithme de hachage) pour le référentiel. Les valeurs valides sont sha1 et (si activé) sha256. sha1 est la valeur par défaut.

  • --template=(répertoire du modèle)

    Spécifie le répertoire à partir duquel les modèles seront utilisés.

  • --separate-git-dir=(git-dir)

    Au lieu d'initialiser le dépôt comme un répertoire dans $GIT_DIR ou ./.git/, créez un fichier texte contenant le chemin vers le dépôt réel. Ce fichier agit comme un lien symbolique Git, agnostique au système de fichiers, vers le dépôt.

  • -b (branch-name)
    --initial-branch=(branch-name)

    Utiliser le nom spécifié pour la branche initiale dans le référentiel nouvellement créé. S'il n'est pas spécifié, le nom par défaut sera utilisé (actuellement master).
