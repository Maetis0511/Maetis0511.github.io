git clone

Utilité

git clone permet de cloner un dépôt dans un nouveau répertoire.

Commande

    
git clone [--template=(template-directory)]
          [-l] [-s] [--no-hardlinks] [-q] [-n] [--bare] [--mirror]
          [-o (name)] [-b (name)] [-u (upload-pack)] [--reference (repository)]
          [--dissociate] [--separate-git-dir (git-dir)]
          [--depth (depth)] [--[no-]single-branch] [--no-tags]
          [--recurse-submodules[=(pathspec)]] [--[no-]shallow-submodules]
          [--[no-]remote-submodules] [--jobs (n)] [--sparse] [--[no-]reject-shallow]
          [--filter=(filter)] [--] (repository)
          [(directory)]

Description

La commande git clone clone un dépôt dans un répertoire nouvellement créé, crée une branche de suivi à distance pour chaque branche du dépôt cloné et crée et extrait une branche initiale qui est dupliquée depuis la branche active actuelle du dépôt cloné.

Options

  • -q
    --quiet

    Mode silencieux. L’avancement n’est pas affiché sur le flux d’erreur standard.

  • -o (name)
    --origin (name)

    Au lieu d’utiliser le nom de distant origin pour suivre le dépôt amont, utilise (name).

  • -b (name)
    --branch (name)

    Au lieu de pointer la HEAD nouvellement créée sur la branche pointée par la HEAD du dépôt cloné, pointe plutôt sur la branche (name). Dans un dépôt non-nu, c’est la branche qui sera extraite.
