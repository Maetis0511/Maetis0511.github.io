git revert

Utilité

git revert permet de revenir sur certains commits existants.

Commande

    
git revert [--[no-]edit] [-n] [-m parent-number] [-s] [-S[(keyid)]] (commit)…​
git revert (--continue | --skip | --abort | --quit)

Description

La commande git revert est utilisée pour enregistrer de nouveaux commits afin d'inverser l'effet de certains commits précédents.

Options

  • -e
    --edit

    Avec cette option, git revert vous laissera éditer le message de commit avant de commiter le revert. C'est l'option par défaut si vous exécutez la commande depuis un terminal.

  • --continue

    Continue l'opération en cours en utilisant les informations dans .git/sequencer. Peut être utilisé pour continuer après avoir résolu les conflits dans un cherry-pick ou un revert qui a échoué.

  • --skip

    Saute le commit courant et continue avec le reste de la séquence.

  • --quit

    Oublie l'opération courante en cours. Peut être utilisé pour effacer l'état du séquenceur après l'échec d'un cherry-pick ou d'un revert.

  • --abort

    Annule l'opération et retourne à l'état pré-séquence.
